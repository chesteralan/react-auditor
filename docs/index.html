react-auditor

A blazing-fast Rust CLI that scans your JS/TS/React codebase for best practices, quality, security, performance, and accessibility issues.

43 rules Rust + oxc MIT license 5 platforms

Quick Install

# Via npm (recommended)
$ npm install -g react-auditor
# Via Cargo
$ cargo install react-auditor
# Scan your project
$ react-auditor

Rule Categories (43 rules total)

Code Quality

No unused vars, no shadowing, consistent returns, complexity caps, and more.

13 rules

React

Hooks rules, missing keys, inline styles, effect deps, setState safety.

13 rules

TypeScript

No any, no non-null assertions, strict null checks, explicit return types.

8 rules

Security

No eval, no dangerouslySetInnerHTML, no hardcoded secrets, no script URLs.

6 rules

Accessibility

img alt, button type, label associations, heading levels, valid ARIA attributes.

5 rules

Performance

Prefer fragments, no bind in JSX, lazy loading, no heavy computation in render.

4 rules

Usage

# Scan with default settings
$ react-auditor

# Output as JSON
$ react-auditor src/ --format json

# Run specific categories
$ react-auditor --rules react,typescript,security

# Auto-fix where supported
$ react-auditor --fix

# Fail CI on more than 10 warnings
$ react-auditor --max-warnings 10

# Write JSON log
$ react-auditor --log audit.json

Pre-commit Integration

Add to your package.json for automatic linting on every commit: 

{
  "lint-staged": {
    "*.{js,jsx,ts,tsx}": [
      "react-auditor --quiet --max-warnings 0"
    ]
  }
}
